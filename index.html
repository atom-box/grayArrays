Functional Javascript

Possibly of interest to none but me.

"Pure functions are easier to reason about. Testing is easier, and pure functions lend themselves well to techniques like property-based testing. Debugging is easier. Programs are more bulletproof. " - Alvin Alexander.

  • The best way to make a webpage reliable is probably to write it in something approaching Haskell, like using the Elm compiler. IBM, Ford Motor Company, and especially No Red Ink have all done this with their sites. This would require migrating to an entirely new language and paradigm, something similar to React Redox.
  • The next best way is Typescript . A major trend of 2017 Web Development was the broad acceptance of Typescript as sort of a poor-mans version of getting the goodness of Functional Programming without shaking up your team too much.
  • The lowest-hanging fruit of all, with a high benefit/cost ratio is functional array methods. These were added to plain vanilla JS for ES5.
